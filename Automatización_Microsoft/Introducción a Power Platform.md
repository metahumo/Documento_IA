# Automatización en Microsoft 365

## Introducción

Microsoft 365 incluye múltiples licencias y herramientas. Todas ellas podemos utilizarlas en nuestro entorno de trabajo; seguramente ya estamos familiarizados y usamos diariamente algunas como email, Excel, Forms, entre otras. Sin embargo, existen otras herramientas que detallaremos aquí, como **Power Automate** y **Power Apps**.  

La idea es que, gracias a estas dos últimas, podemos crear una multitud de automatismos y casos de uso con aquellas herramientas que ya usamos y que requieren de tiempo y presencia, agilizando tareas repetitivas.  

Todas estas tareas manuales, como agendar reuniones o organizar documentación adjunta en carpetas adecuadas, conllevan no solo tiempo, sino también un factor de error humano: olvidos, despistes o realizar acciones de forma rápida pero incorrecta. La automatización puede minimizar estos errores y aportar un **factor de calidad en nuestras tareas**, incrementando nuestra productividad individual.

---

## ¿Qué es Power Platform?

**Power Platform** es un conjunto de herramientas de Microsoft diseñado para facilitar la creación de soluciones empresariales y automatización de procesos sin necesidad de programación avanzada. Incluye herramientas como:

- **Power Automate**: Permite automatizar flujos de trabajo entre diferentes aplicaciones y servicios, eliminando tareas repetitivas y facilitando la integración de herramientas como Outlook, SharePoint, Teams o Forms.  
- **Power Apps**: Nos permite crear aplicaciones personalizadas para necesidades específicas, adaptando procesos internos y recogiendo datos de manera eficiente.  
- **Power BI**: Herramienta de análisis y visualización de datos que complementa los flujos y apps construidos con las otras herramientas.  

Con Power Platform podemos transformar procesos manuales en soluciones automáticas y eficientes, aumentando la productividad y reduciendo errores humanos.

## Conceptos clave

Al trabajar con Power Platform y sus herramientas, es importante conocer algunos conceptos fundamentales:

- **Trigger / Disparador**: Evento que inicia un flujo o acción automatizada (por ejemplo, la recepción de un email o la creación de un formulario).  
- **Action / Acción**: Paso o tarea que se realiza una vez que se activa un trigger (por ejemplo, enviar un correo o crear un registro en SharePoint).  
- **Conectores**: Elementos que permiten integrar diferentes aplicaciones y servicios, facilitando la comunicación entre ellos.  
- **Canvas**: Entorno de diseño visual en Power Apps donde creamos y organizamos nuestras aplicaciones.  
- **Azure**: Plataforma en la nube que proporciona infraestructura y servicios seguros para nuestras aplicaciones y automatizaciones.

## Ciberseguridad

Aunque Power Platform y Microsoft 365 ofrecen mecanismos robustos de seguridad, es fundamental tener en cuenta buenas prácticas:

- **Seguridad de la plataforma**: Azure garantiza autenticación, control de accesos, cifrado de datos y monitorización continua.  
- **Factor humano**: Incluso la herramienta más segura puede generar riesgos si se ejecuta incorrectamente. Es crucial seguir procedimientos y revisar cada flujo o app antes de su despliegue.  
- **Buenas prácticas de trabajo seguro**:  
  - Usar siempre correo corporativo y cuentas autorizadas.  
  - Evitar compartir información sensible con herramientas externas no autorizadas (por ejemplo, ChatGPT, Gemini, etc.) para prevenir filtraciones de datos.  
  - Revisar permisos y conectores utilizados en cada flujo o aplicación.

## Ejemplo de uso

Como ejemplo introductorio, podemos crear un flujo automatizado para **enviar correos o mensajes en Teams** automáticamente ante ciertos eventos, como la recepción de un formulario o la actualización de un documento.  

Otras posibles automatizaciones útiles para una organización como una ONG incluyen:  

- Avisar al responsable de turno y al usuario correspondiente sobre formaciones obligatorias pendientes.  
- Recordatorios automáticos sobre vencimiento de documentación importante.  
- Notificaciones a equipos cuando se generan nuevos registros en bases de datos o formularios.  
- Automatización de la clasificación y archivo de documentos adjuntos recibidos por correo.  
- Creación de informes automáticos de actividades o asistencia a talleres.
